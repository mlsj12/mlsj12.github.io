Coding the Rainbow!

I figured out how to make the rainbow fade in a way that makes it look more "rainbow-y." It took a little math and knowing how coding languages work. I had the idea that the color from before had to be part of the next color's fade to look correctly. I first didn't have any of the colors turn off, so it was ending by fading into white (all the colors turned on). I then realized I needed to turn them off, but I turned them off abruptly half way through the next color's fade.
To accurately fade, the last color needs to fade out. Knowing how programming languages work, I took the variable i and subtracted it against 255 (the highest a color can fade to) to reverse the last color at the same time as the new color grows more stronger.

void setup() {
  // pin 2 is connected to the button to get if the button is pressed or not
  pinMode(2, INPUT);
  // pin 3 is connected to the RED LED
  pinMode(3, OUTPUT);
  // pin 5 is connected to the blue LED
  pinMode(5, OUTPUT);
  // pin 6 is connected to the green LED
  pinMode(6, OUTPUT);
 }
 
 
 void loop() {
  // save the status of button as a variable
   int button_state = digitalRead(2);
 
 
  // when the button is pushed, the led will fade the rainbow
  if (button_state == HIGH) {
    // fade to red
    for (int i = 0; i < 255; i++) {
      analogWrite(3, i);
      delay(25);
      // fade blue out
      analogWrite(5, 255 - i);
      delay(25);
    }
    // fade to green
    for (int i = 0; i < 255; i++) {
      analogWrite(6, i);
      delay(25);
      // fade red out
      analogWrite(3, 255 - i);
      delay(25);
    }
    // fade to blue
    for (int i = 0; i < 255; i++) {
      analogWrite(5, i);
      delay(25);
      // fade green out
      analogWrite(6, 255 - i);
      delay(25);
    }
  }
  // if the button isn't pushed, the light will blink the three colors
  else {
    // red blink
    digitalWrite(3, HIGH);
    delay(500);
    digitalWrite(3, LOW);
    delay(500);
 
 
    // green blink
    digitalWrite(6, HIGH);
    delay(500);
    digitalWrite(6, LOW);
    delay(500);
 
 
    // blue blink
    digitalWrite(5, HIGH);
    delay(500);
    digitalWrite(5, LOW);
    delay(500);
  }
 }

2: Given your schematic, circuit, and firmware, assuming the only thing that draws current is your LEDs, how long would your circuit run if powered by a 1200 mAh battery?

For my circuit, there are a few things happening that are needed to calculate how long it could run on a 1200mAh battery. The first is if the button is clicked, there is a fade of each color either in or out and when it is not clicked, each of the red, blue, green LED is blinking every half a second. Addtionally, there is a resistor limiting the current to 20mA or less.
When the button is pressed, there is usually two LEDs on at once, either fading in or out. Because of this, I am going to assume there is all power for a single LED to make it easier to calcuate the current (taking the same numbers of milliampere as calcuated below).

When the ciruit blinks each light is fully on for that half a second and then off again, meaning when the blinking happens the current is also at it's highest point for each of the lights. With the 220Ω resistor for red and green LED, they are using 5V - 1.8V = 220Ω × A ---> A = 0.01454 ---> 14.54mA. With the 100Ω resistor for the blue LED, it is using 5V - 3.3V = 100Ω × A ---> A = 0.017 ---> 17mA.

To find the length of time this circut can run on a 1200mAh battery, I am going to take the assumption that the fade and the blink will happen each half the time. With this, I only need to divide milliampere per hour by milliamperes of current. Runtime = 1200mAh / ((17mA + 14.54mA)/2) = 76h. I averaged the two currents for red, green, and blue to make it easier to calcuate. So 76 hours is an estimate of the actual runtime on a 1200mAh battery with fading and blinking happening each 50% of the time.
